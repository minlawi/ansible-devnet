## SSH Key Authentication for Cisco Network Devices

### Method 1: Manual Setup

#### Generate SSH Keypair for Network Devices

```bash
ssh-keygen
```

When prompted:
- **File**: `/home/lawi/.ssh/cisco_rsa`
- **Passphrase**: Press Enter (empty for no passphrase)

#### Add SSH RSA Support to Client Configuration

Add this line to `/etc/ssh/ssh_config`:

```bash
PubkeyAcceptedKeyTypes +ssh-rsa
```

#### Get MD5 Hash of Public Key

Cisco IOS uses the MD5 hash format:

```bash
ssh-keygen -lf ~/.ssh/cisco_rsa.pub -E md5
```

Example output:
```
3072 MD5:f7:bf:98:6f:8e:82:f4:24:bb:26:0a:6d:f1:88:e2:e7 lawi@dev (RSA)
```

Note: Use the hash **without colons** in the configuration: `f7bf986f8e82f424bb260a6df188e2e7`

#### Configure User and SSH Key on Network Device

```bash
configure terminal
ip ssh pubkey-chain
username ansible_admin
key-hash ssh-rsa f7bf986f8e82f424bb260a6df188e2e7 lawi@dev
exit
exit
end
write memory
```

**Important:** Replace the hash and comment (`lawi@dev`) with your actual values.

### Method 2: Ansible Automation

#### Generate SSH Keypair (Same as Method 1)

```bash
ssh-keygen
```

#### Configure SSH Client (Same as Method 1)

Add `PubkeyAcceptedKeyTypes +ssh-rsa` to `/etc/ssh/ssh_config`

#### Create User and Configure SSH Key Using Ansible

**Default user (ansible_admin):**
```bash
ansible-playbook -i inventory/cisco_devices.ini 03_ssh_key_auth/configure_ssh_key_auth.yml
```

**Custom user (e.g., lawi, network_admin):**
```bash
ansible-playbook -i inventory/cisco_devices.ini 03_ssh_key_auth/configure_ssh_key_auth.yml -e "target_user=lawi"
ansible-playbook -i inventory/cisco_devices.ini 03_ssh_key_auth/configure_ssh_key_auth.yml -e "target_user=network_admin"
```

The playbook will:
1. Check if the user exists on network devices
2. Create the user with privilege level 15 if needed
3. Calculate the MD5 hash of your SSH public key
4. Configure the key using `key-hash` format

#### Check if User and SSH Key Are Configured

**Check default user:**
```bash
ansible-playbook -i inventory/cisco_devices.ini 03_ssh_key_auth/check_ssh_key_auth.yml
```

**Check custom user:**
```bash
ansible-playbook -i inventory/cisco_devices.ini 03_ssh_key_auth/check_ssh_key_auth.yml -e "target_user=lawi"
```

#### Remove User and SSH Key Configuration

**Remove default user:**
```bash
ansible-playbook -i inventory/cisco_devices.ini 03_ssh_key_auth/remove_ssh_key_auth.yml
```

**Remove custom user:**
```bash
ansible-playbook -i inventory/cisco_devices.ini 03_ssh_key_auth/remove_ssh_key_auth.yml -e "target_user=lawi"
```

#### Test SSH Key Authentication

```bash
# Connect as ansible_admin
ssh -i ~/.ssh/cisco_rsa ansible_admin@<device_ip_address>

# Connect as custom user
ssh -i ~/.ssh/cisco_rsa lawi@<device_ip_address>
```

### Configuration Format Reference

**Manual Configuration Structure:**
```
ip ssh pubkey-chain
  username <username>
   key-hash ssh-rsa <md5-hash> <comment>
```

**Example:**
```
ip ssh pubkey-chain
  username ansible_admin
   key-hash ssh-rsa f7bf986f8e82f424bb260a6df188e2e7 lawi@dev
```

**Note:** Cisco IOS stores SSH public keys using their MD5 hash, not the full key string.
